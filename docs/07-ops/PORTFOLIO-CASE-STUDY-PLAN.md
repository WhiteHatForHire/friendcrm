# PORTFOLIO-CASE-STUDY-PLAN.md - Symposium Studios Portfolio Plan

This plan packages Friend CRM as a Symposium Studios portfolio/product item.

The positioning should make the tension explicit:

> Friend CRM is a tongue-in-cheek retro relationship desk with modern local-first functionality, AI-aware workflows, and privacy-first product boundaries.

The surface can be goofy, bureaucratic, and fake-serious. The product thinking underneath should feel current, careful, and unusually deliberate.

---

# Portfolio Goal

Create a page on the Symposium Studios site that presents Friend CRM as:

- A memorable product concept.
- A playable prototype.
- A design system experiment.
- An AI product ethics case study.
- A proof of engineering and QA discipline.

The page should not make the app look like a normal business tool. It should feel like a retro intelligence-desk parody that happens to solve a real human problem: remembering people thoughtfully.

---

# Core Positioning

## Short Pitch

Friend CRM is a private, local-first relationship memory desk for people who swear they will remember.

## Portfolio Pitch

Friend CRM turns personal context into a fake-serious retro command desk: part early-2000s web satire, part private notebook, part AI-aware workflow prototype. It helps a person remember context, promises, boundaries, and next moves while deliberately avoiding the creepy parts: scraping private messages, automated outreach, hidden scoring, and unreviewed AI memory.

# Symposium Studios Page Structure

## 1. Hero

Purpose: make the product instantly legible and visually sticky.

Content:

- Product name: Friend CRM
- Subhead: A tongue-in-cheek retro relationship desk with modern AI-aware workflows.
- Hero image: desktop People screen or People + rail screenshot.
- CTAs:
  - View Demo
  - View GitHub
  - Read Case Study

Design notes:

- Use the loudest, cleanest desktop screenshot.
- Let the retro UI carry the personality.
- Add a small label such as "Local-first prototype / fake-data demo".

## 2. Product Snapshot

Purpose: explain what it does in one scan.

Bullets:

- Capture notes about people and conversations.
- Review source-backed suggested memories and open loops.
- Generate pre-meeting briefs and next-move drafts.
- Organize next moves on a Plot Board.
- Export, restore, and delete user-owned data.
- Run a public-safe fake-data demo without real private data.

## 3. The Product Joke

Purpose: explain why the style is intentional.

Suggested copy:

Friend CRM borrows the language of fake private bureaus, overconfident early web dashboards, and satirical relationship surveillance. The joke is on the interface, not the user. The product looks like it might be creepy, then carefully refuses the actually creepy behavior.

Show:

- Sidebar / branded shell screenshot.
- Poster Lab screenshot.
- Cheeky copy examples.

## 4. The Real Product Constraint

Purpose: show mature product judgment.

Suggested copy:

The central design problem was not "can AI remember things?" It was "what should a personal relationship tool refuse to do?" Friend CRM uses source-backed suggestions, explicit user review, local-first storage, and export/delete paths so durable memory remains user-controlled.

Principles:

- No private message scraping.
- No automated outreach.
- No hidden scoring.
- No durable AI-generated memory without user confirmation.
- User-owned data remains exportable and deletable.

## 5. Product Walkthrough

Purpose: show the demo flow.

Sections:

1. People Desk - scan the relationship file.
2. Dossier Rail - inspect context, memories, open loops, and next moves.
3. Debrief Booth - capture a note.
4. Review Panel - approve only source-backed suggestions.
5. Plot Board - move next actions through planning states.
6. Poster Lab - generate fake dossier art for fun, not memory.
7. Evidence Locker - export, import, restore, and reset.
8. Mobile / Tablet - prove the system works outside desktop.

Use the screenshot gallery from `docs/07-ops/portfolio-screenshots-2026-06-29/`.

## 6. Engineering Notes

Purpose: make the implementation credible.

Highlights:

- Vite, React, TypeScript.
- Local browser storage for the prototype.
- Schema-versioned JSON export/import.
- Deterministic fallback behavior for demo safety.
- Server-side AI boundary for provider-backed development routes.
- Vitest coverage.
- Browser, mobile, and tablet regression scripts.
- Synthetic trial harness with fake data.

## 7. Design Evolution

Purpose: show taste and iteration.

Arc:

- Started as a practical private desk.
- UI felt too sterile.
- Direction shifted toward retro web satire and fake intelligence-bureau humor.
- Visual language became louder: sticker headings, glossy controls, classified panels, hover motion, dossier art.
- Usability stayed grounded through browser testing and mobile/tablet passes.

## 8. Postmortem

Purpose: make the work reflective, not just flashy.

Suggested points:

- Humor can make a sensitive product feel less clinical, but trust copy still has to be plain.
- Review/confirmation is more important than raw generation.
- Browser QA caught layout problems that unit tests would not catch.
- The product line between useful memory and surveillance has to be guarded in every feature.
- The strongest portfolio angle is the contrast: fake-serious retro shell, serious privacy and AI behavior underneath.

## 9. Current Status

Suggested copy:

Friend CRM is a playable local-first prototype with a fake-data demo, browser regression suite, mobile/tablet coverage, source-backed AI workflow shells, and export/restore paths. It is ready to package as a hosted deterministic demo. Production hosting, authentication, and real user data workflows are intentionally out of scope for the portfolio demo.

# Ready-To-Paste Captions

## People Desk

The main desk turns a private contact list into a fake-serious social intelligence file, complete with warmth, trust, context, memories, and open loops.

## Brief

Briefs are generated from local context and framed as drafts, not truth. The system treats AI as a helper, not an authority.

## Review Panel

Suggested memories and open loops must be reviewed before becoming durable records, keeping the user in control of what becomes "official."

## Plot Board

The Plot Board turns social follow-through into a playful planning surface without automated sending or hidden pressure mechanics.

## Poster Lab

Poster Lab pushes the retro parody all the way: fake dossier art for comedy, with clear boundaries that it is not evidence, analysis, or memory.

## Evidence Locker

Export, restore, reset, and local storage controls keep the prototype honest about user-owned data.

## Mobile

The mobile view keeps the product usable as a compact private desk rather than a desktop layout squeezed into a phone.
